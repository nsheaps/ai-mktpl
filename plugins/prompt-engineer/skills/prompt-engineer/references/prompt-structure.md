# Prompt Structure Template

This is the canonical structure for prompts generated by the prompt-engineer skill.
Every generated `.claude/prompts/init.md` MUST follow this structure. Sections marked
(required) cannot be omitted. Sections marked (if applicable) are included when relevant.

---

## Table of Contents for Generated Prompt

```
1.  Project Overview (required)
2.  Architecture (required)
3.  Monorepo / Package Structure (required)
4.  Feature Specifications & Detailed Feature Sections (required)
5.  End-to-End Test Plan (required)
6.  UI/UX Guidelines (if applicable — skip for CLIs, libraries)
7.  Data Model / Schema (if applicable)
8.  API Design (if applicable)
9.  Security & Auth (if applicable)
10. Deployment & CI/CD (required)
11. Claude Code Integration (required — this is the agent infrastructure)
12. Task Breakdown (required — the master task list)
13. Final Notes (required)
```

### 3. Monorepo / Package Structure

```markdown
## 3. Monorepo Structure

packages/
├── core/ # Business logic, no UI, no platform code
├── ui/ # UI components (if applicable)
├── web/ # Web entry point (if applicable)
├── cli/ # CLI entry point (if applicable)
├── server/ # Server entry point (if applicable)
└── shared/ # Shared types, utilities
```

Adapt this to the project. Not everything needs a monorepo — a single-package project
is fine for CLIs or small tools. But the structure must be clear.

## Writing Guidelines

1. **Be specific, not vague.** "Implement a REST API" is bad. "Implement POST /api/notes
   that accepts {title, body, tags[]} and returns the created note with id and timestamps"
   is good.

2. **Every feature needs acceptance criteria.** If you can't test it, you can't verify it.

3. **Task granularity matters.** Each task should be 30-90 minutes of agent work. If a task
   feels like "implement the entire database layer," split it.

4. **Phase -1 is always the same pattern.** Bootstrap Claude Code config, install plugins,
   create TASKS.md, push to git. The only things that change are project-specific details.

5. **The Claude Code Integration section (11) is ~40% of the prompt by size.** This is
   intentional. The agent infrastructure IS the product — features are built on top of it.

6. **Include non-goals.** Telling the agent what NOT to build is as important as what to build.
   Without non-goals, the agent will scope-creep.

7. **Reference the user's exact words** when describing features. Don't paraphrase away
   important nuance.
